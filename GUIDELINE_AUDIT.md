# 햇켓 과제 가이드라인 최종 점검

점검 기준: 플랫폼 기능, `secure_coding_checklist.csv`, 제출 요구사항

## 1. 기능 요구사항

| 항목 | 상태 | 구현 내용 |
|---|---|---|
| 회원가입·로그인·로그아웃 | 완료 | 비밀번호 해시, 세션 인증, POST 로그아웃 |
| 프로필 관리 | 완료 | 닉네임·소개글·로그인 비밀번호·결제 비밀번호 |
| 상품 관리 | 완료 | 등록·조회·수정·삭제, 소유자 검사, 이미지 최대 5장 |
| 검색·필터 | 완료 | 키워드·카테고리·판매 상태·정렬 |
| 찜 | 완료 | 등록·해제·목록 조회 |
| 전체 채팅 | 완료 | 인증, 길이 검증, 차단 필터, 사용자별 3초 제한 |
| 1:1 채팅 | 완료 | 접근권한, 차단 검사, 사용자별 1초 제한 |
| 친구·차단 | 완료 | 친구 요청·처리·삭제, 사용자 차단·해제 |
| 신고 | 완료 | 상품·사용자 신고, 대상별 묶음 검토, 승인·기각 |
| 송금·상품 결제 | 완료 | 잔액·결제 비밀번호 검증, 트랜잭션, 중복 결제 차단 |
| 관리자 | 완료 | 사용자 정지, 상품 숨김, 신고 검토, 송금·감사 로그 |

## 2. 보안 체크리스트

### 적용 완료

- 모든 HTML POST 폼 CSRF 보호
- 로그인 비밀번호 및 결제 비밀번호 해시 저장
- HttpOnly·SameSite 세션 쿠키, 2시간 세션 만료
- HTTPS 환경에서 Secure 쿠키 활성화 가능
- 상품·신고·프로필·채팅 입력값 서버 검증
- 상품 수정·삭제 시 소유자 검사
- 채팅방 접근권한 및 차단 관계 검사
- 실제 이미지 포맷 검증, 크기 제한, 안전한 파일명 생성
- 로그인 5회 실패 시 5분 잠금
- 전체 채팅 3초·1:1 채팅 1초 Rate Limiting
- 사용자별 한 시간 최대 5건 신고
- 동일 상품·사용자의 여러 신고를 묶어서 검토
- 관리자 승인·기각, 처리 메모·처리자·처리 시각 저장
- 승인 시 사용자 정지 또는 상품 숨김
- SQLite 파라미터 바인딩
- 서버 측 상품 가격 결정 및 송금 트랜잭션
- CSP, X-Frame-Options, X-Content-Type-Options 등 보안 헤더
- 사용자용 400·403·404·405·413·500 오류 페이지

### 운영 환경에서 별도로 적용할 항목

1. HTTPS 및 WSS
2. `COOKIE_SECURE=1`
3. 다중 서버 환경의 중앙 Rate Limiting 저장소(Redis 등)
4. SQLite DB 파일의 최소 권한 설정 또는 운영용 DBMS 전환
