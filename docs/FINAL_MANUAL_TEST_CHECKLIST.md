# 햇켓 최종 수동 테스트 체크리스트

최종 제출 직전에 서로 다른 일반 사용자 계정 2개와 관리자 계정 1개로 확인한다.

## 1. 인증 및 권한

- [ ] 회원가입 후 로그인 가능
- [ ] 잘못된 비밀번호 5회 입력 시 5분 잠금
- [ ] 로그아웃 버튼으로 정상 로그아웃
- [ ] 비로그인 상태에서 상품 등록 페이지 접근 차단
- [ ] 일반 사용자의 `/admin` 접근 차단

## 2. 상품

- [ ] 이미지 없는 상품 등록 가능
- [ ] 정상 이미지가 포함된 상품 등록 가능
- [ ] 이미지가 아닌 파일 업로드 차단
- [ ] 상품 수정 및 삭제 가능
- [ ] 다른 사용자의 상품 수정·삭제 차단
- [ ] 검색·카테고리·상태·정렬 필터 동작
- [ ] 찜 등록·해제 및 목록 반영

## 3. 송금

- [ ] 결제 비밀번호 설정 가능
- [ ] 결제 비밀번호 없이 송금 차단
- [ ] 잘못된 결제 비밀번호 입력 시 송금 차단
- [ ] 본인 상품 결제 차단
- [ ] 잔액 초과 송금 차단
- [ ] 정상 송금 시 송신자·수신자 잔액 반영
- [ ] 상품 결제 후 판매 완료 처리
- [ ] 같은 상품 중복 결제 차단
- [ ] 송금 내역에 상대방과 상품 정보 표시

## 4. 채팅·친구·차단

- [ ] 전체 채팅 송수신
- [ ] 전체 채팅 3초 이내 연속 전송 제한
- [ ] 상품에서 판매자와 1:1 채팅방 생성
- [ ] 1:1 채팅 송수신 및 읽지 않은 메시지 반영
- [ ] 다른 사용자의 채팅방 URL 직접 접근 차단
- [ ] 친구 요청·수락·거절·삭제
- [ ] 사용자 차단 후 전체·1:1 채팅 차단
- [ ] 차단 해제 후 기능 복구

## 5. 신고 및 관리자 검토

- [ ] 상품 상세에서 상품 신고 접수
- [ ] 공개 프로필에서 사용자 신고 접수
- [ ] 신고자가 본인을 신고하지 못함
- [ ] 같은 신고자가 동일 대상을 중복 신고하지 못함
- [ ] 사용자별 한 시간 5건 초과 신고 차단
- [ ] 동일 상품의 여러 신고가 관리자 화면에서 한 묶음으로 표시
- [ ] 동일 사용자의 여러 신고가 관리자 화면에서 한 묶음으로 표시
- [ ] 묶음 안에서 신고자·사유·접수 시각 확인
- [ ] 상품 신고 묶음 승인 시 해당 상품 숨김
- [ ] 사용자 신고 묶음 승인 시 해당 사용자 정지
- [ ] 신고 묶음 기각 시 대상에 제재가 적용되지 않음
- [ ] 관리자 메모·처리자·처리 시각 저장
- [ ] 관리자 감사 로그에 일괄 처리 건수와 조치 내용 기록

## 6. UI 및 오류 처리

- [ ] 로고와 헤더 메뉴가 깨지지 않음
- [ ] 상품 등록·대시보드·프로필 화면 가운데 정렬
- [ ] 상품 상세의 관심 상품·송금·채팅 버튼 줄바꿈 없음
- [ ] 공개 프로필 아바타가 원형 안에 표시
- [ ] 모바일 폭에서 주요 화면 가로 스크롤 없음
- [ ] 존재하지 않는 주소에서 사용자용 404 페이지 표시
- [ ] GET 방식 `/logout` 접근 시 사용자용 405 페이지 표시
